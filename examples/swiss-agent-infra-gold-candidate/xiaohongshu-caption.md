# 小红书标题

2026 上半年，Agent 基础设施真正变厚了

# 正文

2026 上半年看 Agent，我越来越觉得：真正值得看的变化，不只是模型又强了多少，而是 Agent 基础设施本身在演进。

更具体地说，模型外面那层系统正在变厚。

工具怎么接、权限怎么收、上下文怎么调度、失败怎么恢复、轨迹怎么审计、人什么时候介入，这些问题开始决定 Agent 能不能从 demo 进入生产。

我把文章里最重要的 5 个判断压成了这组卡：

1. Agent 基础设施变厚  
重点不是某个模型榜，而是模型外面的运行时系统。

2. 从模型到部署  
产品动作开始围绕岗位角色、任务状态和部署证明展开。

3. Harness 是边界  
它不是提示词和工具的外壳，而是把 Agent 能做什么、不能做什么写进系统里。

4. API / CLI / MCP 不是站队  
先看 Agent 要接哪里、权限怎么管、结果要不要进上下文。

5. 企业先分任务，再上 Agent  
简单任务一次模型调用就够；路径稳定用 workflow；步骤开放再上 Agent。

我现在会更关注：谁能把 Agent 放进真实组织和真实系统里稳定运行，而不是谁又发了一个漂亮 demo。
