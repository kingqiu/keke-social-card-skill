# 小红书标题

别只盯模型榜，Agent 真正变厚的是运行时

# 备选标题

1. 2026 上半年，Agent 基础设施变了什么？
2. 为什么 Agent 从 demo 到生产这么难？
3. 企业落地 Agent，先别急着多 Agent

# 正文

2026 上半年看 Agent，我最大的感受是：

别只盯模型榜。

真正影响企业落地的，正在变成模型外面的那一整套系统：工具、权限、上下文、状态、审计、恢复、评估，以及人怎么介入。

我把这条线概括成一句话：

Agent 周围的运行时正在变厚。

几个变化很明显：

1. 竞争从模型能力转向部署能力  
角色、流程、协作界面、任务状态，开始比单次回答更重要。

2. Harness 不再只是胶水  
它负责工具、权限、边界和爆炸半径，不能只靠 prompt 兜底。

3. 上下文管理变成信息调度  
重点不是窗口有多大，而是什么信息该常驻、按需加载或留在工具里。

4. Runtime 开始成为分水岭  
长期运行、状态恢复、观测、审计、多租户，才是从 demo 到生产的关键。

5. 企业别急着多 Agent  
简单任务一个模型调用就够；路径稳定的任务用 workflow；步骤开放、需要动态判断时，再上 Agent。

下半年我会重点看：谁能把 Agent 放进组织和真实系统里稳定运行，而不只是发一个漂亮 demo。
