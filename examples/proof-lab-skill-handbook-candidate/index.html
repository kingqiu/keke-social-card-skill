DEFINITION · WHEN TO LOAD

Skill 管的是
特定场景的经验

AGENTS.md 像工作态度规范，Skill 像某个岗位的岗前培训。一个管通用标准，一个管具体怎么做。

AGENTS.md

写文件、搜索、复盘、协作方式，这些是所有任务都要遵守的底层规矩。

Skill

日报格式、代码规范、行业术语、验收清单，这些是某类任务才需要的经验。

DESIGN MAP · FIVE PATTERNS

先想清楚：
你要 AI 学会什么

用工具、按模板生成、做质量评审、先问清楚问题、按流程推进，这五类需求对应五种 Skill 设计模式。

模式不是装饰名词，而是决定 Skill 该放工具、模板、清单、问题，还是流程。

别先写内容

先判断任务本质，否则 Skill 会变成杂乱资料夹。

每种模式

都在回答同一个问题：AI 在这个场景里到底该怎么行动。

Choose the pattern before filling the folder03
PATTERN 01 · TOOL WRAPPER

工具包装器：
只在需要时加载专业工具

适合 FastAPI、特定框架、内部 CLI 这类重复专业任务。关键是触发词精准，太宽会误触发，太窄又用不上。

WHEN重复调用特定框架、CLI 或内部工具。 RISK触发词太宽会误触发，太窄又用不上。 OUTPUT只加载必要工具知识，用完退出上下文。

解决什么

不用每次把一整套工具规范重新塞进对话。

设计重点

明确何时触发、用哪些命令、失败时怎么退回。

Load the tool only when the task asks for it04
PATTERN 02 · GENERATOR

生成器：
把输出变稳定

如果你总嫌 AI 每次写出来格式不一样，就先定义模板和风格指南，让它像填空一样完成结果。

WHEN结果格式必须稳定，不能每次自由发挥。 RISK只给风格词，不给字段和缺失信息处理。 OUTPUT模板、字段、追问和验收一起固化。

适合场景

会议纪要、API 文档、项目周报、固定格式内容。

设计重点

模板、字段、缺失信息追问、输出验收规则都要写清楚。

Template first, then fill the missing fields05
PATTERN 03 · REVIEWER

评审员：
把“质量好不好”变成清单

评审型 Skill 不负责替你做事，它负责检查结果有没有达标。模糊评价没用，最好写成具体的是非题。

WHEN你已有结果，需要 AI 帮你判断质量。 RISK用“整体评价”这种模糊标准。 OUTPUT把质量拆成可复用的是非题清单。

解决什么

把“感觉不对”变成可复用、可执行的检查标准。

设计重点

每条标准都要能判断：通过、失败、需要补充证据。

A reusable checklist beats vague judgment06
PATTERN 04 · INVERSION

反转：
让 AI 先问清楚再动手

很多返工不是能力问题，而是 AI 一听开头就开始猜。反转模式把 AI 变成面试官，先补齐背景、目标和限制。

WHEN任务背景不完整，贸然动手容易返工。 RISKAI 为了显得聪明，提前猜你的意图。 OUTPUT先提问补齐背景、目标和约束。

解决什么

避免基于不完整信息输出一堆看似正确的废话。

设计重点

定义必须追问的问题，以及什么时候信息足够可以开始。

Ask before acting, especially under uncertainty07
PATTERN 05 · PIPELINE

管道：
让多步骤任务不能跳步

有些任务天然要分阶段推进。管道型 Skill 要写清每一步做什么、产物是什么、没通过时不能进入下一步。

WHEN任务天然分阶段，中间步骤不能跳。 RISKAI 直接给结论，省掉关键检查。 OUTPUT每一步都有输入、产物、验收和失败处理。

解决什么

AI 没被约束时容易省略中间步骤，直接跳到结论。

设计重点

每一步都要有输入、输出、验收条件和失败处理。

A process with gates is easier to trust08
OPTIMIZATION · AUTO RESEARCH

Skill 也需要
自动迭代

当通过率卡住时，手动调 prompt 很慢。Auto Research 的思路是让 AI 按检查清单反复测试、微调、保留有效改动。

检查清单

最好是 3-6 个具体、可衡量的是非题。

验证方式

每轮只改一点，看结果有没有变好；好就保留，不好就撤销。

Iteration needs measurable pass/fail checks09
FUTURE · SELF IMPROVING SKILLS

最后回到一个问题：
你想让 AI 学会什么

好的 Skill 不靠堆知识，而靠把经验组织成结构，让它能被调用、被评估、被组合，最终自己变得更好。

自我评估Skill 要知道这次输出质量如何，而不是完全依赖外部评价。
记住改进如果能保存失败原因和改动历史，下一轮就不用从零开始。
协作组合工具包装器、评审员、管道可以各司其职，比单个 Skill 硬扛更像真实工作。
少而精花时间打磨一个真正好用的 Skill，比装一百个用不上的更有价值。
核心判断如果某个 Skill 用起来鸡肋，先检查触发词是否准确、内容是否太杂、标准是否能被执行。
Organize experience so AI can call it at the right moment10
