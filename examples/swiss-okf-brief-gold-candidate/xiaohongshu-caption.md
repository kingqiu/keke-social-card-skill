# 小红书标题

Google 推出的 OKF，可能是 Agent 知识交付的新信号

# 正文

Google Cloud 最近推了一个 Open Knowledge Format，简称 OKF。

我觉得它最值得看的地方，不在于 Google 又发布了什么新产品，而在于它把“组织知识怎么交给 AI Agent”这件事，往开放格式的方向推了一步。

很多 Agent 项目真正卡住的点，常常不是模型能力或工具数量，而是上下文太散：指标定义在文档里，表结构在数据目录里，业务口径在聊天记录里，runbook 又在另一套系统里。Agent 想回答一个具体问题，往往要临时跨系统拼知识。

OKF 的设计很克制：用普通文件目录表达层级，用 Markdown 写正文，用 YAML frontmatter 放少量可查询字段，再用普通链接表达概念关系。

也就是说，表、指标、数据集、API、playbook、runbook，都可以变成一个个可读、可检索、可版本控制的知识文件。人能打开看，Agent 也能直接解析；可以放进 Git，可以审查变更，也可以被不同工具消费。

这件事对 Agent 开发者的意义是：知识不再只能困在某个产品、某套 SDK 或某个厂商的知识平台后面。真正重要的，可能是把知识上下文做成一层可迁移、可审计、可共同维护的“基础设施”。

OKF 未必会马上成为事实标准，但它释放的信号很明确：AI Agent 要进入真实工作流，光会调用工具还不够，它还需要一种稳定的方式去理解组织知识。

你觉得 Agent 时代，组织知识更应该放在知识库里，还是更应该像代码一样管理？
