# 小红书标题

2026 上半年，Agent 基础设施正在变厚

# 正文

最近看 Agent，我越来越不想只看“哪个模型又强了”。

真正值得看的，是 2026 上半年 Agent 基础设施发生了什么变化：当 Agent 从 demo 走向生产，模型外面的系统正在变厚。

我的核心判断是：

Agent 的难点正在从“会不会回答”，转向“能不能被部署、被约束、被追踪、被恢复、被组织接住”。

这组卡按原文主线拆成 10 张：

1. Agent 基础设施变厚  
运行时不是某个产品名，而是工具、权限、上下文、状态、审计、恢复、评估和人机介入组成的系统外壳。

2. 从模型到部署  
竞争开始从模型能力，转向岗位角色、任务状态、协作界面和部署证明。

3. FDE 重新变热  
企业 AI 的瓶颈不只是有没有模型，而是模型怎么进入复杂组织的生产流程。

4. Harness 是边界  
安全不能只靠 prompt，而要写进产品和工程系统。

5. API / CLI / MCP 不是站队  
先看 Agent 要接哪里、权限怎么管、结果要不要进上下文。

6. 上下文变成信息调度  
重点不是窗口有多大，而是什么信息常驻、按需加载、压缩或留给工具。

7. 生产需要 Runtime  
长期运行、状态恢复、多租户、观测和审计，才是 demo 到生产之间的底座。

8. Loop 的重点是反馈  
没有验证的循环，只是在反复生成。

9. 企业先分任务，再上 Agent  
简单任务用模型调用，路径稳定用 workflow，步骤开放再用 Agent。

10. 下半年看稳定运行  
真正值得跟踪的，是谁能把 Agent 放进组织和真实系统里稳定运行。

我觉得这也是下半年看 AI 产品和 Agent 平台时很重要的一条线：不要只看 demo 漂不漂亮，要看它有没有把权限、上下文、工具、状态、审计和反馈这些脏活做扎实。

你最近看 Agent 产品时，最关心的是模型能力，还是部署和工程化能力？欢迎评论区一起聊。
