# Gold Example Inputs

Use this when creating a `Gold` example or a publishing-quality user package.

## Minimum Inputs

Ask for these only when they are missing and materially affect quality:

- Product/tool name and the exact public name to show.
- One-sentence value promise: what the reader can do after seeing this post.
- Target reader: role, gender/lifestyle context when relevant, skill level, and pain point.
- Real proof assets: screenshots, output examples, before/after, photos, or product result images.
- Three strongest use scenarios, written in the user's natural language.
- Must-keep terms, banned terms, account tone, and fixed hashtags.

## Best Assets For Gold Quality

For AI tool launch or tutorial examples:

- 1 cover-level proof screenshot or result image.
- 2-4 workflow screenshots showing input, process, output, or before/after.
- 1 real output artifact users can inspect.
- A short note explaining what changed after using the tool.

For lifestyle, outfit, beauty, food, home, product, or gear examples:

- 1 complete result image.
- 3-6 detail images or item photos.
- Scene constraints: weather, budget, body/comfort concerns, taste, occasion, or substitution needs.
- Why each choice matters, not only what each item is.

## If Inputs Are Missing

Do not label the example `Gold`.

Use:

- `Prototype` when the example proves layout or workflow only.
- `Silver` when structure is strong but proof assets are incomplete.
- `Gold` only when the result is publishable and evidence is credible.

## Ask The User

Use a compact request:

```text
要做到 Gold 级，我最需要这 4 类素材：
1. 产品/Skill 的真实运行截图或输出图
2. 你最想主打的一句话价值承诺
3. 目标用户和 3 个典型使用场景
4. 必须保留/不能说的词，以及固定标签
```

If the user cannot provide assets, proceed with a clearly marked `Prototype` or `Silver` example and record the gap in `QA.md`.
