# Xiaohongshu Copywriting Guidelines

This file guides Xiaohongshu titles, captions, hashtags, and QA notes.

Official platform rules can change. When a task is high-risk or the user asks for compliance certainty, browse and verify current Xiaohongshu official creator/community/commercial-content rules before finalizing.

Content ecology baseline: avoid clickbait where the title seriously mismatches content, exaggerated claims, fake authority, fabricated data, and misleading marketing.

## Deliverables

Create:

- `xiaohongshu-caption.md`: main title, body, hashtags, publishing notes.
- `copy-variants.md`: 3-5 title options, opening options, CTA/comment prompts, alternate tags.

## Title

Rules:

- Lead with user benefit, scene, result, or pain point.
- Keep it concrete and human.
- Avoid unsupported absolutes: `最`, `第一`, `必`, `100%`, `全网`.
- Avoid low-quality bait: `震惊`, `爆火`, `不看后悔`.
- Match the image promise.

Patterns:

- 痛点型：`每天纠结穿什么的人有救了`
- 结果型：`一句话，让 AI 出 3 套穿搭`
- 体验型：`我做了个随时能问的穿衣搭子`
- 对比型：`以前翻衣柜，现在直接问搭子`
- 场景型：`通勤/约会/周末，穿搭不用硬想`

## Caption Body

Recommended structure:

1. Real pain or scene in 1-2 sentences.
2. What this post/tool/content is.
3. 3-5 concrete use cases or takeaways.
4. How to use it or what result to expect.
5. Honest limitation if relevant.
6. Comment prompt asking for scenes, feedback, or experience.

Avoid:

- Fake user feedback.
- Fake stats.
- Fake endorsements.
- Over-promising results.
- Platform-external hard sell.
- Repeating card text word for word.

## Hashtags

Use 5-8 tags:

- 1-2 domain tags.
- 1-2 scenario tags.
- 1-2 method/tool tags.
- optional account-specific tag.

Avoid stuffing unrelated hot tags.

## Copy QA

Check:

- Is the title inflated?
- Does caption invent facts?
- Does it involve medical, financial, legal, or other high-risk claims?
- Are rankings, numbers, endorsements, and results sourced?
- Does it create misleading platform-external conversion pressure?
- Does it match the rendered images?
